# Domain Pack: X / Twitter Content Operations

Use this pack for X/Twitter content planning, article reconstruction, tone adjustment, thread writing, and removing obvious AI style.

## Primary objectives

- Turn ideas, articles, product updates, or research into platform-native posts.
- Preserve the original insight while making the language human, direct, and readable.
- Avoid generic AI phrasing, fake authority, and empty motivational language.
- Build a repeatable content loop: idea → draft → style edit → publish candidate → performance review → next iteration.

## Typical tasks

- Rewrite an article into a thread.
- Convert formal writing into spoken, direct language.
- Remove AI smell.
- Generate hooks and variants.
- Extract key claims and turn them into posts.
- Localize content for a target audience.
- Build a posting calendar.
- Analyze performance and topic clusters.

## Style rules

Prefer:

- Concrete nouns and verbs.
- Shorter sentences with occasional natural variation.
- Specific examples.
- Tension, contrast, and practical insight.
- A clear point of view.

Avoid:

- “In today’s fast-paced world”.
- Over-polished transitions.
- Excessive bullet lists.
- “Unlock”, “revolutionize”, “game-changer” unless intentionally used.
- Fake personal experience.
- Claiming data or results without evidence.

## Rewrite protocol

1. Extract the core thesis.
2. Identify the intended reader.
3. Decide the post type: observation, contrarian take, tutorial, story, product update, teardown, or thread.
4. Preserve facts; do not invent experience.
5. Rewrite in the chosen voice.
6. Produce 3-5 hook variants if requested.
7. Run anti-AI smell pass.
8. Return final version plus brief explanation of style choices.

## Report visuals

- Topic cluster map.
- Hook performance table.
- Impression-to-engagement funnel.
- Posting cadence timeline.
- Voice consistency checklist.
