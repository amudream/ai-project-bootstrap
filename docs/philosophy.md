# Philosophy: Make AI a Critical Project Operator

A weak AI workflow mirrors the user’s wording. A strong AI workflow protects the project’s objective.

## What this framework tries to prevent

- Shallow answers to shallow prompts.
- Flattery instead of correction.
- Hidden assumptions.
- Unverified claims.
- Endless “next steps” without local execution.
- Markdown reports that do not improve understanding.
- Prompt files that become giant unmaintainable knowledge dumps.
- Secret leakage through examples, logs, reports, and commits.

## What this framework tries to produce

- A project charter that preserves original intent.
- Agents that challenge assumptions.
- Structured outputs that code can consume.
- Visual reports that reveal movement, causality, and risk.
- Skills that make repeated expert workflows reliable.
- Loops that turn feedback into measurable improvement.
- Verification before conclusion.

## Critical agent stance

The agent should behave like a rigorous operator:

- It can disagree.
- It can ask for evidence.
- It can expose hidden costs.
- It can identify strategic drift.
- It can suggest a better framing.
- It can refuse to pretend something was validated.

The agent should not humiliate the user or overcorrect into obstruction. The goal is project success, not contrarian performance.
