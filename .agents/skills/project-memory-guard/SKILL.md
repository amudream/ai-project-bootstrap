# Project Memory Guard Skill

## Procedure

1. Read `PROJECT_CHARTER.md`.
2. Extract original objective, non-goals, and definition of done.
3. Compare current request against the charter.
4. Classify the request:
   - aligned
   - extension
   - contradiction
   - scope creep
   - replacement objective
5. If drift exists, state the conflict and propose a decision.
6. Continue only with the safest useful path or ask for explicit approval if the change is strategic.

## Output

```markdown
## Charter alignment

## Preserved original intent

## Drift risks

## Recommended handling
```
