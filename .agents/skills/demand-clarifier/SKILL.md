# Demand Clarifier Skill

## Trigger

Use this skill before implementation when the task is ambiguous, broad, strategic, or likely to drift.

## Procedure

1. Read `PROJECT_CHARTER.md` if present.
2. Restate the objective in precise terms.
3. Separate goals, constraints, assumptions, non-goals, and acceptance criteria.
4. Identify missing decisions and contradictions.
5. Challenge weak assumptions.
6. Define MVP and high-quality target.
7. Produce a task breakdown that can be executed or delegated.
8. Update or propose updates to the charter if the project direction changes.

## Output

```markdown
## Clarified objective

## Original intent to preserve

## Constraints

## Missing decisions

## Weak assumptions / challenge points

## MVP

## High-quality version

## Acceptance criteria

## Execution plan
```
