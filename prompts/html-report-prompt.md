# HTML Report Prompt

```text
请生成一个现代 HTML 报告。不要把 markdown 简单套 CSS。

报告必须基于结构化数据，包含：
- KPI 卡片，显示当前值、变化、趋势方向和解释。
- 至少一个时间序列图，展示增长/下跌。
- 至少一个流动图，展示资金、用户、库存、风险或数据如何移动。
- 至少一个热力图或矩阵，展示风险、分群或优先级。
- 一个可排序/可过滤的发现表。
- 一个决策建议区，明确行动、优先级、证据和风险。
- 数据来源、更新时间、假设和限制。

请根据行业调整视觉语言和指标：
- 阿里国际站：曝光、点击、访客、询盘、有效询盘、关键词机会、国家热力、产品漏斗。
- 股票复盘：K 线、成交量、换手率、板块热力、涨停时间线、触发和失效条件。
- 独立站开发：客户匹配矩阵、开发信漏斗、国家/行业分布、异议热力。
- 内容运营：reach、engagement、hook、topic cluster、posting cadence。
- 客服 CRM：需求聚类、异议热力、响应时间、询盘质量漏斗。

输出必须是可直接打开的单文件 HTML，除非我明确允许外部依赖。
```
